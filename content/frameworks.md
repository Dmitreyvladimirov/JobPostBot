# Interview Frameworks — Quick Reference

## Diagnose Framework

**Goal:** What changed → behavioral shift → mechanism → validate

**5 Steps:**
1. What exactly dropped? (volume / depth / quality)
2. Where in user journey? (entry / selection / consumption / action)
3. Problem category: Content/Supply | UX/Flow | Technical | External
4. 1–2 most likely hypotheses (technical = fallback, not default)
5. 2–3 data checks that prove or disprove

**Signal reading:**
- Gradual + existing users → algorithm/recommendation degradation
- Sudden + all users equally → technical or external event
- New users affected more → activation/onboarding failure
- Browse but don't convert → decision/trust problem

**Strong opening:**
> "I'd like to understand what exactly dropped — sessions, depth, or interactions? Was it sudden or gradual? All users or a specific segment?"

**Strong answer structure:**
> "Based on data, the problem is at [journey stage]. Main hypothesis: [X] because it explains [all signals]. [Y] is less likely because [reason]. To validate: 1. [behavioral check] 2. [quality metric] 3. [content/system correlation]"

---

## Universal Product Sense Structure

1. Clarify (segment, use case, metric)
2. Define user segment + core value
3. Map user journey
4. Identify failure points
5. Prioritize ONE problem (impact × frequency × severity)
6. Define exact user failure (behavior → mechanism → impact)
7. 1–2 solutions tied to product philosophy
8. Metrics (primary + guardrail)
9. Tradeoffs

**Product types → core value:**
- Marketplace: match supply/demand
- Content: consume relevant content
- Social: communication/networking
- SaaS: complete workflow efficiently
- Utility: reliable fast action

---

## 8 Question Types

1. **Metric improvement** — improve retention/engagement/conversion → framework above
2. **Open product design** — "design X for Y" → user problems first, not metrics
3. **Feature design** — consider ecosystem + product philosophy + unintended consequences
4. **Favorite product** → why successful → problem → tradeoff → solution → "why now"
5. **Zero to one** — problem → segment → MVP → success metric
6. **Strategy/tradeoff** — benefit → cost → timing → recommendation
7. **Diagnose** — framework above
8. **Execution/ops** — adoption stage → barriers → rollout plan

---

## Prioritization (any case)

Name both options → compare on: impact | frequency | severity | fits philosophy → choose with reasoning → name validation metric.

> "I'd prioritize X over Y because it directly attacks the root cause and affects [N]% of users. I'd validate using [metric]."

---

## Senior Signal Moves

- **"Why now"** — connect to the product's evolution stage
- **One tradeoff** — "the risk is [X], I'd monitor [guardrail metric]"
- **Philosophy fit** — "this fits [product]'s approach because it's [non-invasive / user-controlled]"
- **Hypothesis rejection** — "[Y] is less likely because it would cause [sudden drop / affect all users equally]"
- **Existing features** — "I assume [feature] already exists — I'd focus on improving its effectiveness"
